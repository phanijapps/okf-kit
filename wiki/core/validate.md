# Overview

`validate_bundle(root)` walks the bundle and classifies findings into three severities. Errors block conformance: missing or unparseable frontmatter, an empty `type`, or malformed reserved files. Warnings are non-blocking: missing `title` or `description`, broken internal links, and concept ids that violate the segment regex. Info covers extension keys, nested sub-bundle markers, and `okf_version` status.

The consumer is permissive by design: it never rejects a bundle for missing optional fields, unknown types, extension keys, or broken links. This keeps bundles useful as they grow, get refactored, and are partially generated by agents. The report is a plain dataclass, so it serializes cleanly to JSON for CI gates. See [conformance](/concepts/conformance.md).

# Examples

```
okf validate mykb            # human report; exit 1 if not conformant
okf validate mykb --json     # machine-readable for CI
```
