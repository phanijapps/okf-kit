# Overview

Conformance is the contract that makes an OKF bundle interoperable: a consumer can rely on a small set of structural rules and must tolerate everything else. Per SPEC 9, a bundle is conformant if and only if every non-reserved Markdown file has a parseable YAML frontmatter block, every frontmatter has a non-empty `type`, and reserved files follow their defined structure when present.

The permissive side is just as important: consumers must not reject a bundle for missing optional fields, unknown `type` values, unknown extension keys, broken cross-links, or missing `index.md` files. This keeps bundles useful as they grow, get refactored, and are partially generated by agents.

`okf validate` is the judge: it classifies findings into blocking errors and non-blocking warnings and info, and never raises on malformed input.

# Examples

```
okf validate mykb            # conformant? exit 1 if not
okf validate mykb --json     # for CI
```
